Uma Breve História sobre o Git

Para entendermos como o Git funciona vamos voltar um pouco no tempo como muitas coisas na vida. Desenvolvido pelo mesmo criador do sistema Linux no ano de 2005, Linus Torvalds, o Git se espalhou entre os Devs. Inicialmente, as pessoas desenvolvedoras que trabalhavam no kernel (núcleo) do Linux usavam o sistema do BitKeeper. Porém, os criadores removeram o acesso gratuito após uma polêmica sobre direitos autorais. Insatisfeito, Torvalds decidiu criar um sistema semelhante com código aberto, dando origem assim ao Git. E, só por curiosidade, a palavra Git vem de uma gíria usada por ingleses que remete à “cabeça dura”, ou seja, pessoas que acham que sempre têm razão.

Comandos Básicos

  • ls - Listar Sistema, todas as pastas e localidades da nossa maquina.
  • ctrl + L = Limpa a tela de códigos.
  • ctrl + ↑ (seta) = Faz com que apareça o histórico de comandos que já havíamos utilizado.
  • cd = Acessar o diretório (Acessar a Pasta).
  • cd .. = Voltar para a pasta anterior.
  • cd ./ = Adicionamos todo o caminho para onde queremos ir.
  • mkdir = Cria uma pasta
  • pwd = Nos mostra onde estamos da nossa maquina.

E agora?

Após abrir o seu git bash devemos aplicar algumas configurações que iremos iniciar agora.

Configurações iniciais

  • git config --global user.name “matheus” = Para configurar e registrar o nome do usuário dentro do git.
  • git config --global user.email “email” = Assim registramos um e-mail no git.
  • git config --global init.defautbranch = Definimos qual será a raiz do nosso repositório, main ou master. (Já vou explicar o que seria a Branch)

Depois de realizarmos esses comandos dentro do git podemos iniciar o nosso reposítorio. Mas antes preciso te explicar o que seria uma Branch de forma simples claro!

  • git branch = São versões do desenvolvimento, onde podemos acompanhar cada feito do projeto.

Iniciando os Trabalhos

  • git init = Iniciamos o controle do git dentro da pagina (Fazendo o monitoramento da pagina), toda e qualquer alteração a partir de agora ficara registrado.
  • git remote add origin : (sempre será usado) para dicionar o endereço remoto (endereço do github)
  • git remote --v : ele serve para mostrar a qual repositório remoto está ligado (configurando remoto o repositório ligado)
  • git remote rm origin : remover a ligação entre o local e o remot
  • git add "nome do arquivo"txt = para o git enxergar as mudanças no arquivo
  • git add . = adiciona tudo que está na página do repositorio
  • git commit (ponto de restauração) - comitar tudo que foi feito usando um editor passando a mensagem (quase nunca usar)
  • git commit -m "alteração do novoArquivo.txt" : permite passar uma mensagem com informação de alteração detalhada
  • git commit --amend -m : para corrigir (quando errar) o commit anterior
  • git log -h - Vai mostrar todas as funções que aquele comando oferece, por exemplo, o log.
  • git log = Mostra quem alterou, quando alterou e oque alterou do repositorio e aparece o commit feito.
  • git log --oneline : ele mostra somente o que foi feito sem se importar com data e quem fez.
  • git log --author “nome ou email da pessoa“ : serve para saber o que a pessoa em especifico fez. obs: ele é sensível a caixa alta
  • git log --grep “inicial” : ele pesquisa na mensagem especifica que você buscar em todos commits
  • git log shortlog - mostra de forma resumida todo o log

obs: origin/master, origin/HEAD (é o que está no github)
obs: origin (remoto) head -> master (local)
